wxmap-templatemsg

小程序模板消息

下发条件说明：

  1. 支付
    当用户在小程序内完成过支付行为，可允许开发者向用户在7天内推送有限条数的模板消息（1次支付可下发3条，多次支付下发条数独立，互相不影响）
  2. 提交表单
    当用户在小程序内发生过提交表单行为且该表单声明为要发模板消息的，开发者需要向用户提供服务时，可允许开发者向用户在7天内推送有限条数的模板消息（1次提交表单可下发1条，多次提交下发条数独立，相互不影响）

注意事项（前端）：formID：

  • 支付场景下为prepay_id 。
  • 非支付场景下为form表单id。非广义id，需开启form表单属性report-submit=’true’，即声明为消息模板发送form表单。
  • 开发环境formId正确值为 the formId is a mock one 。
  • 测试消息模板必须使用真机环境，否则formid错误，错误码errer40128 ，Post请求下发失败。

注意事项（后端）：

  • access_token 2小时限时，超过时限需重新请求accesstoken，否则post下发模板请求错误。

后台可用模板: 25个/小程序。

自定义模板：
可以申请添加新模板，审核通过后可使用。 模板审核说明

实现：

1、小程序后台新加入一条测试模板：

选用测试员模板，勾选两项返回值：测试人、备注

2、小程序内新建index.wxml
form表单 开启模板推送report-submit=true

<form bindsubmit="formSubmit" bindreset="formReset" report-submit='true' >
    <view class="section">
        <view class="section__title">手机号</view>
        <input name="testphone" placeholder="手机号" />
    </view>
    <view class="btn-area">
        <button formType="submit">提交</button>
        <button formType="reset">重置</button>
    </view>
</form>

3、index.js POST请求

4、开启在线调试

5、真机 扫码小程序 打开表单提交后获取 服务通知。结束。

CentOS7 Nginx Let's Encrpty

关于let’s encrpty

Let’s encrpty是一个免费的，自动化的，开放的证书颁发机构。于2015年三季度推出的数字证书认证，将通过旨在消除当前手动创建和安装证书的复杂过程的自动化流程，为安全网站提供免费的SSL/TLS证书。
Let’s encrypt 维基百科
GitHub代码库
官方文档Let’s encrypt

官方推荐使用 Certbot 进行证书的颁发安装。所以下一步安装官方工具certbot。

certbot安装。

下文以：

* webroot：example/html/blog 
* 域名：example.com 
* 邮箱：example@example.com 
* / 根路径

为例：

①创建目录

mkdir letsencrypt
cd letsencrypt

②通过git下载certbot代码库

git clone https://github.com/certbot/certbot.git

③执行安装命令 注意替换example

certbot certonly --webroot -w /example/html/blog -d example.com -d www.example.com -m example@example.com --agree-tos

–agree-tos官方说明
【The first time you run the command, it will make an account, and ask for an email and agreement to the Let’s Encrypt Subscriber Agreement; you can automate those with –email and –agree-tos】
④安装完毕 提示
⑤证书位置

/etc/letsencrypt/live/example.com/

⑥生成DHE参数
关于DHE出处：
– 所有的nginx版本在往Diffiel-Hellman输入参数时依赖OpenSSL。
不幸的时，这就意味着Ephemeral Diffiel-Hellman（DHE）会使用OpenSSL的这一缺陷，包括一个1024位的交换密钥。由于我们正在使用一个2048位的证书，DHE客户端比非ephemeral客户端将使用一个更弱的密钥交换。–

Nginx配置

nginx 配置文件

根据nginx安装路径 配置nginx.conf文件 例:/etc/nginx/conf.d/excample.conf

centos7防火墙开启端口

firewall-cmd --zone=public --add-port=443/tcp --permanent
sudo firewall-cmd --reload
